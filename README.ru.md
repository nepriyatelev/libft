## Описание

Этот проект посвящён созданию собственной C-библиотеки с полезными, универсальными функциями для будущих C‑проектов в Школе 42. Цель — собрать полноценную библиотеку (libft.a), которая переосмысливает стандартные функции C и дополняется утилитарными функциями. Реализуя их самостоятельно, вы разберётесь, как устроены функции стандартной библиотеки, углубите понимание C, управления памятью и структур данных (в частности, связных списков).

Библиотека libft станет основой для всех дальнейших проектов 42, так как многие задания будут зависеть от неё и использовать эти функции.

## Инструкции

### Компиляция

```bash
make all       # Сборка библиотеки (создаёт libft.a) — явное правило
make           # Сборка библиотеки (правило по умолчанию, эквивалентно 'make all')
make clean     # Удалить объектные файлы (.o)
make fclean    # Удалить объектные файлы и библиотеку (.a)
make re        # Полная пересборка (эквивалент 'make fclean' затем 'make all')
make bonus     # Сборка с бонусными функциями

```

### Установка

Чтобы использовать libft в своих проектах:

```bash
# Скопируйте папку библиотеки в ваш проект
cp -r libft /path/to/your/project/

# В вашем Makefile добавьте:
LIBFT_DIR = ./libft
LIBFT = $(LIBFT_DIR)/libft.a

$(LIBFT):
    make -C $(LIBFT_DIR)

# Линковка вашей программы с libft
your_program: $(LIBFT) your_objs.o
    cc -I $(LIBFT_DIR)/includes -L $(LIBFT_DIR) your_objs.o -lft -o your_program
```

## Подробное описание библиотеки

### Часть 1: Функции libc (25 функций)

#### Работа со строками (11 функций)
- `ft_strlen` — длина строки
- `ft_strcpy` — копирование строки (небезопасно)
- `ft_strncpy` — копирование не более n символов строки
- `ft_strcat` — конкатенация строк (небезопасно)
- `ft_strncat` — конкатенация не более n символов
- `ft_strcmp` — сравнение двух строк
- `ft_strncmp` — сравнение не более n символов
- `ft_strchr` — поиск первого вхождения символа
- `ft_strrchr` — поиск последнего вхождения символа
- `ft_strstr` — поиск первого вхождения подстроки
- `ft_strnstr` — поиск первого вхождения подстроки в первых n байтах

#### Операции с памятью (6 функций)
- `ft_memset` — заполнение памяти константным значением
- `ft_bzero` — обнуление памяти
- `ft_memcpy` — копирование памяти (без перекрытия)
- `ft_memmove` — копирование памяти (возможны перекрытия)
- `ft_memcmp` — сравнение областей памяти
- `ft_memchr` — поиск байта в памяти

#### Проверка символов (6 функций)
- `ft_isalpha` — буква ли символ
- `ft_isdigit` — цифра ли символ
- `ft_isalnum` — буквенно-цифровой ли символ
- `ft_isascii` — входит ли в ASCII
- `ft_isprint` — печатаемый ли символ
- `ft_toupper`/`ft_tolower` — изменение регистра

#### Функции конвертации (2 функции)
- `ft_atoi` — строка → целое число
- `ft_calloc` — выделение памяти с инициализацией нулями

### Часть 2: Дополнительные функции (11 функций)

- `ft_substr` — извлечь подстроку
- `ft_strjoin` — конкатенация двух строк с выделением памяти
- `ft_strtrim` — убрать указанные символы в начале и конце
- `ft_split` — разбиение строки по разделителю
- `ft_itoa` — целое число → строка
- `ft_strmapi` — применить функцию к каждому символу с индексом
- `ft_striteri` — применить функцию к каждому символу (мутабельно)
- `ft_putchar_fd` — вывод символа в файловый дескриптор
- `ft_putstr_fd` — вывод строки в файловый дескриптор
- `ft_putendl_fd` — вывод строки с переводом строки
- `ft_putnbr_fd` — вывод целого числа

### Часть 3: Функции для связного списка (9 функций)

```c
typedef struct s_list
{
    void            *content;  // Универсальное хранение данных
    struct s_list   *next;     // Указатель на следующий элемент

```

- `ft_lstnew` — создать новый узел списка
- `ft_lstadd_front` — добавить узел в начало
- `ft_lstadd_back` — добавить узел в конец
- `ft_lstsize` — количество узлов
- `ft_lstlast` — последний узел списка
- `ft_lstdelone` — удалить и освободить один узел
- `ft_lstclear` — удалить и освободить весь список
- `ft_lstiter` — применить функцию к каждому узлу
- `ft_lstmap` — создать новый список, применив функцию к каждому узлу

## Требования

- **Язык:** C
- **Компилятор:** cc с флагами -Wall -Wextra -Werror
- **Norm:** соответствие стандарту 42 School
  - Максимум 25 строк на функцию
  - Максимум 5 функций в файле
  - Максимум 4 локальные переменные на функцию
  - Стиль C89 (все переменные объявляются в начале функции)
  - Максимум 80 символов в строке
- **Память:** без утечек, корректная обработка ошибок
- **Сборка:** Makefile с правилами: all, clean, fclean, re, bonus
- **Зависимости:** только стандартная библиотека C
